## Document Organization and Hierarchy

Create a clear structure for your docs using folders and pages. You nest pages indefinitely to match your project's architecture, like `/api/reference/endpoints` or `/guides/user-setup`.

Use the drag-and-drop interface to reorder content. Each page supports rich MDX formatting, including components like `<Callout>` for notes.

| Level | Example Path | Use Case |
|-------|--------------|----------|
| Root | `/introduction` | Landing page |
| Folder | `/api` | API section |
| Page | `/api/auth` | Authentication guide |
| Nested | `/api/auth/oauth` | OAuth details |

<Callout kind="tip">
  Prefix page slugs with numbers like `01-introduction.mdx` for automatic sorting.
</Callout>

## Version History and Tracking

Every edit creates a snapshot. You view diffs, restore previous versions, or compare branches.

<Expandable title="Advanced Version Controls" default-open="false">
  Access history via the page menu. Filter by date or author.

  ```javascript
  // API to fetch versions
  const versions = await fetch('https://api.example.com/docs/{pageId}/versions');
  console.log(versions); // Array of snapshots
  ```
</Expandable>

<CodeGroup tabs="CLI,API">
  ```bash
  peter versions list --page introduction.mdx
  peter versions revert --id abc123
  ```
  ```javascript
  // Restore via API
  await fetch('https://api.example.com/docs/{pageId}/revert', {
    method: 'POST',
    body: JSON.stringify({ versionId: 'abc123' })
  });
  ```
</CodeGroup>

## Search Functionality and Filters

Peter Parker's search indexes all content instantly. You type queries and refine with filters.

Key filters:

- **By tag**: `tag:api`
- **By date**: `after:2024-01-01`
- **By author**: `author:john@example.com`
